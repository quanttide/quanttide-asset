# 飞书 Wiki 资产目录规范

---

## 核心问题

飞书 Wiki 的目录结构有什么规范？如何验证这些规范？

---

## 1. 目录结构规范

### 1.1 树形结构

所有 Space 的目录都是一棵树：

```
Space（空间）
└── Root（根节点，depth=0）
    ├── Container（分类节点，has_child=true）
    │   ├── Container（子分类）
    │   │   └── Leaf（叶子文档）
    │   └── Leaf
    └── Leaf
```

### 1.2 节点类型

| 类型 | 特征 | obj_type | 说明 |
|------|------|----------|------|
| Root | parent_token="" | docx/folder | 根节点，Space 入口 |
| Container | has_child=true | docx/folder | 分类节点，用于组织结构 |
| Leaf | has_child=false | docx/file | 叶子节点，存储实际内容 |

### 1.3 元数据字段

```json
{
  "node_token": "节点唯一标识",
  "title": "节点标题",
  "obj_type": "docx | file | folder",
  "parent_token": "父节点 token（根节点为空）",
  "has_child": true | false
}
```

## 2. Space 类型与分类模式

观察了多个 Space，总结出 4 种分类模式：

### 2.1 固定分类模式

**Space**：议事档案

```
议事档案
├── 议题
│   ├── 议事规则
│   │   ├── 2025年第49周-议事规则1-辩论
│   │   └── 2025年第49周-议事规则2-决议
│   ├── 议程
│   │   └── 2025年第49周-议程2-新媒体运营
│   ├── 决策
│   └── 计划
```

**规范**：
- 分类名固定：议题/议程/决策/计划/提案/审计/澄清/评估/谈判/报告/决议
- 叶子文档命名：`{年份}年第{周}周-{类型}{序号}-{主题}`

### 2.2 列表分类模式

**Space**：协作档案

```
协作档案
├── 首页
├── 任务管理
│   ├── 任务列表
│   │   ├── 2025年第9周-任务1-优化制冰机提示词.docx
│   │   └── 2025年第9周-任务2-优化制冰机数据报告.docx
│   └── 飞书任务管理教程
```

**规范**：
- 分类名动态：如"任务管理"、"任务列表"
- 叶子文档命名：`{年份}年第{周}周-任务{序号}-{主题}.docx`

### 2.3 主题分类模式

**Space**：图书馆

```
图书馆
└── 编程语言
    ├── Android
    │   └── 第一行代码 Android.pdf
    ├── C Language
    │   └── ANSI C Standard.pdf
    └── Flutter
        └── Flutter实战.pdf
```

**规范**：
- 按主题分类：编程语言/Android/C/Flutter/Java...
- 叶子节点是文件：书名.pdf

### 2.4 功能分类模式

**Space**：软件库

```
软件库
├── 开发者装机必备工具.docx
├── SSH软件
│   └── putty.exe
└── Python及IDE
    ├── pycharm-community-2022.1.3.exe
    └── python-3.10.11-amd64.exe
```

**规范**：
- 功能分类：SSH软件/Python及IDE
- 叶子节点是安装包：.exe/.pdf

## 3. 目录规范总结

### 3.1 通用规范（所有 Space 必须遵守）

| 规范 | 说明 | 验证级别 |
|------|------|----------|
| 必须有根节点 | depth=0, parent_token="" | error |
| 根节点不能有子节点 | has_child=false | error |
| Container 必须有子节点 | has_child=true 时实际必须有子节点 | error |
| 父子关系完整 | parent_token 指向存在的节点 | error |
| 节点标题不能为空 | title ≠ "" | error |

### 3.2 类型特定规范（按 Space 类型可选验证）

| Space 类型 | 分类模式 | 叶子文档命名 | 验证级别 |
|------------|----------|--------------|----------|
| 议事档案 | 固定分类 | `YYYY年第N周-类型-主题` | warning |
| 协作档案 | 列表分类 | `YYYY年第N周-任务K-主题.docx` | warning |
| 图书馆 | 主题分类 | `书名.pdf` | warning |
| 软件库 | 功能分类 | `软件名.exe` | warning |

## 4. 契约设计

### 4.1 通用契约（所有 Space 适用）

```yaml
# contract.yaml

spec_version: 0.0.1

# 通用验证规则
validation:
  rules:

    # 结构规则
    - name: 必须有根节点
      assert: node_exists(depth=0)
      level: error

    - name: 根节点不能有子节点
      assert: node(depth=0).has_child == false
      level: error

    - name: Container 必须有子节点
      description: has_child=true 的节点必须有实际子节点
      assert: all(node.has_child == (child_count > 0) for node in containers)
      level: error

    - name: 父子关系完整性
      assert: all_parent_tokens_valid()
      level: error

    # 命名规则
    - name: 节点标题不能为空
      assert: all(node.title != "" for node in nodes)
      level: error
```

### 4.2 Space 特定契约（可选）

```yaml
# contract_space_shanyidangan.yaml

space_name: 议事档案
space_id: "7564357713069883393"

# 特定规则
validation:
  rules:

    - name: 一级分类必须是固定值
      assert: |
       一级分类 in ["议题", "议程", "决策", "计划", "提案", "审计", "澄清", "评估", "谈判", "报告", "决议"]
      level: error

    - name: 叶子文档命名格式
      assert: matches(node.title, "^\\d{4}年第\\d{1,2}周-.+")
      where: node.depth >= 2 and node.has_child == false
      level: warning
```

## 5. 验证流程

```
┌─────────────────────────────────────────────────────────────┐
│                      验证执行流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 扫描发现                                                │
│     └→ catalog.py 扫描 Space，获取 metadata.json             │
│                                                             │
│  2. 加载契约                                                │
│     └→ 读取通用规则 + Space 特定规则（可选）                 │
│                                                             │
│  3. 构建资产树                                              │
│     └→ 根据 parent_token 构建树形结构                        │
│                                                             │
│  4. 执行验证                                                │
│     └→ 遍历规则，执行断言                                   │
│                                                             │
│  5. 输出报告                                                │
│     └→ 差异报告 + 统计信息                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
